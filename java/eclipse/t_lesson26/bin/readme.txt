异常的处理流程
使用try和catch来处理异常。try中的代码是可能发生异常的代码，如果在try中发现了异常，会跳离try语句块，并且去寻找catch中的异常，如果找到之后就会执行catch中的代码，
执行完成之后不会再回到try中，而是程序继续向下执行。如果没有找到catch，假设是RuntimeException，会继续抛出。

异常的捕获
异常的捕获是支持多态的，如果只捕获Exception类的异常，就相当于捕获了所有的异常，异常的捕获也支持写多个catch，但是需要注意的是，catch时会顺序匹配
如果先匹配到父类的异常，后面的catch都不会执行，所以子类的异常必须在父类异常之前捕获，否则编译报错。
