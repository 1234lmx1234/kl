IO流
Input Output
输入流使用InputStream
输出流使用OutputStream
输入和输出都是从程序的角度来看，如果要向一个文件写内容使用的输出流，如果要读取一个文件中的内容，使用输入流

节点流和过滤流
节点流和过滤流一样都有输入流和输出流
节点流指的是可以直接用来创建对象的流
过滤流指的是用来修饰节点流，让节点流拥有更多的功能，所以过滤流不能直接创建，要在节点流的基础上创建。

字符流和字节流
只要是类的结尾使用Stream表示的都是字节流，使用Reader或者Writer表示的都是字符流
字符流主要用来处理字符信息。

转换流
主要用于将字节流转换为字符流。流里面传的数据一定要是字符数据。在很多的方法中，诸如网络编程，得到的数据都是字节流对象，而
网络聊天基本都是字符数据，此时就可以使用转换流
转换流一般都是OutputStreamWriter和InputStreamReader来表示的
如果希望从标准输入中获取数据，就需要使用转换流

总结：
IO流主要用来处理文件内容的读写
InpuStream和OuputStream主要用来处理字节流，如图片，docx，xlsx
输入和输出的关系主要以程序为中心，如果要读取文件，是读取文件中的内容到程序，应该使用输入流，要写文件需要使用输出流


节点流和过滤流
节点流可以独立创建对象0，File流都是节点流，过滤流都是用来增强节点流功能的，Data,Object,Buffered这些都是过滤流
不管节点流还是过滤流，只要关闭最外层即可

字节流和字符流
字节流用来处理字节数据，类都是以Stream结尾，而字符流都是以Writer或者Reader结尾，一般来说，输入的字符流都会使用
BufferedReader进行处理，因为该类有readLine方法，字符的输出流一般使用PrintWriter来处理。

转换流
将字节流转换为字符流。
